# TideCompass 测试问题记录

更新时间：2026-05-06

## 1. 登录失败（真机体验版）

### 现象

- 体验版扫码进入后，点击登录提示失败。

### 已定位原因

- `users` 集合权限规则配置为：
  - `"read": "doc.openId == auth.openid"`
  - `"write": false`
- 由于登录流程中首次用户会执行 `users.add(...)`，`write: false` 会导致新增失败。

### 修复方案

- 将 `users` 集合规则调整为仅允许用户读写自己的文档：

```json
{
  "read": "auth != null && doc.openId == auth.openid",
  "write": "auth != null && doc.openId == auth.openid"
}
```

### 待验证

- 保存并发布规则后，真机重新登录：
  - 是否可成功登录；
  - `users` 集合是否新增当前用户记录；
  - 控制台是否仍出现 `permission denied`。

## 2. 云函数超时风险（已做防护）

### 背景

- 之前出现调用超时/卡住现象，已对 AI 相关云函数增加客户端侧快速超时与降级。

### 已修改

- `cloudfunctions/generateAIReport/index.js`
  - 外部请求超时改为 10s；
  - 增加 `Promise.race` 硬超时兜底。
- `cloudfunctions/parseFile/index.js`
  - 外部请求超时改为 9s；
  - 增加 `Promise.race` 硬超时兜底。

### 待验证

- 重新部署云函数后，真机触发对应流程：
  - 是否仍出现超时；
  - 云函数日志中是否出现新的 timeout 关键词；
  - 失败时是否能快速返回而非长时间卡住。
